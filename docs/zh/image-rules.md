# 生图规则说明

## 当前最小可用标准

当前生图系统采用一套简化但更稳定的规则，目标是：

- 不出现乱码文字图
- 视觉更干净
- 更适合公众号新媒体使用

## 封面图规则

要求：

- 横版，目标压缩后接近 900:500
- 简约高级
- 视觉干净
- 主体突出
- 无文字
- 无水印
- 适合新媒体传播
- 高清质感

## 内文插图规则

要求：

- 默认只生成 1 张正文主图
- 视觉上适配竖版或方形内容思路
- 轻量化
- 有氛围感但不花哨
- 不打扰阅读
- 与整篇文章风格统一

## 固定提示词结构

所有图像 prompt 都采用固定结构：

```text
【主体/画面内容】 + 【场景/氛围】 + 【风格】 + 【色彩】 + 【构图画质】 + 【负面词】
```

## 统一负面词

```text
模糊，低清，水印，文字，杂乱，扭曲，瑕疵，低俗，过度渲染
```

## 为什么这样设计

旧版本图像提示词更像“文章摘要提示词”，容易导致：

- 图片里出现乱码中文文字
- 文字海报感过重
- 画面脏乱
- 封面图和内文图职责不清

当前规则先优先保证：

- 画面不带字
- 可用
- 统一
- 不影响阅读

## 当前已落地

- 封面图固定 prompt 结构：已落地
- 内文图固定 prompt 结构：已落地
- 默认正文图数量 = 1：已落地
